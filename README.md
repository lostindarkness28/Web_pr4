# Система моніторингу вітроелектростанцій (WindPower API & Registry)

Професійний вебдодаток для централізованого моніторингу та реєстрації об'єктів вітрової енергетики. Система дозволяє керувати технічними параметрами станцій, відстежувати покази датчиків у реальному часі та проводити аналітичні розрахунки потужності.

## Основні функції
* **Керування об'єктами (CRUD):** Повний цикл роботи з ВЕС через REST API (Створення, Читання, Оновлення, Видалення).
* **Моніторинг показів:** Відстеження швидкості вітру, обертів ротора та поточної генерації.
* **Автоматична аналітика:** Серверний розрахунок енергоефективності та максимального потенціалу турбін.
* **Middleware Logging:** Вбудована система логування кожного запиту для аудиту безпеки.
* **Адаптивний інтерфейс:** Сучасний Glassmorphism дизайн для зручного керування з будь-яких пристроїв.

## Технологічний стек
* **Backend:** Node.js, Express.js (REST API)
* **Frontend:** JavaScript (Fetch API), HTML5, CSS3
* **Database:** JSON-based persistent storage
* **Testing:** Postman, Automated Testing Scripts

## Структура проєкту
* `server.js` — Головний файл сервера, логіка API та Middleware.
* `public/` — Статичні файли клієнтської частини (інтерфейс моніторингу).
* `data/wind-plants.json` — Локальна база даних у форматі JSON.
* `scripts/test-api.js` — Скрипт для автоматизованого тестування всіх ендпоінтів.

##  API Endpoints (Документація)
| Метод  | Ендпоінт                   | Опис                                   |
| :----- | :------------------------- | :------------------------------------- |
| GET    | `/api/wind-plants`         | Отримати список усіх ВЕС               |
| GET    | `/api/wind-plants/:id`     | Детальні дані конкретної станції       |
| POST   | `/api/wind-plants`         | Реєстрація нового енергооб'єкта        |
| PATCH  | `/api/wind-plants/:id`     | Оновлення показів датчиків (вітер/кВт) |
| DELETE | `/api/wind-plants/:id`     | Видалення об'єкта з реєстру            |
